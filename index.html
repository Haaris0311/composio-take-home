Key Patterns: What the Data Shows

The Headline

  • 74 of 100 apps are buildable today with standard auth, documented APIs, and self-serve credentials. The "easy wins" dominate.
  • OAuth2 is king (48% of apps), followed by API keys (40%). These two patterns cover ~88% of all auth scenarios.
  • Developer/Productivity tools are the easiest category - 100% ready-today, 100% self-serve-free. Zero friction.
  • AI/Research is the hardest - 50% have no public API at all. These are too new or too closed.
  • Finance is gated by design - 50% require sales contact or customer status. Regulatory reality.
  • The #1 blocker is "no API exists" (6 apps), followed by "contact-sales gate" (12 apps). Technical debt is rare; business decisions dominate.

Pattern 1: Auth Follows Category

CRM/Sales: OAuth2 dominates (90%). Developer tools: API keys + OAuth2 in equal measure. Finance: API keys preferred (60%). Communications: bot tokens as a distinct pattern (Telegram, Slack, Discord).

Pattern 2: Self-Serve Correlates with Buildability

Of the 54 self-serve-free apps, 51 (94%) are ready-today. Of the 12 contact-sales apps, only 3 are buildable. Access friction IS the blocker.

Pattern 3: Enterprise = Gated, Startup = Open

Established enterprise tools (Salesforce Commerce, DealCloud, PitchBook) gate behind sales. Newer startups (Twenty, Firecrawl, Apify) default to open APIs.

Pattern 4: MCP Servers Follow Developer Mindshare

Apps with existing MCP servers are disproportionately developer-facing (GitHub, Slack, Notion) or high-volume integrations (Salesforce, HubSpot). Long tail is untouched.

Pattern 5: The "API Gap" Is Shrinking

Only 7/100 apps have no API at all (7%). Most are AI-native tools too new to have built one. The rest all have at least REST endpoints. GraphQL adoption is ~15% and growing.

Pattern 6: Easy Wins vs Outreach Needed

Easy wins (74 apps): Developer + Productivity + Communications + most CRM. Needs outreach (13 apps): Enterprise ecommerce, finance, ad platforms. Wait for API (13 apps): AI-native, niche tools.

Methodology

Stratified sample of 20 apps (2 per category) verified by an independent AI agent + human spot-check. The verifier was instructed to be strict: only "correct" if confident, "corrected" for actual errors, "uncertain" if can't verify.

Findings from Verification

Honest Misses

  • Pylon, Pumble: Too new/niche for confident API documentation claims. Flagged as "uncertain."
  • fanbasis, higgsfield, iPayX: Minimal public presence. "Not feasible" may be conservative if they have undocumented APIs.
  • Salesforce Commerce Cloud vs Salesforce: Agent initially conflated; corrected to note separate auth flows.
  • MCP server claims: "Community-built" is hard to verify exhaustively. Some may exist on GitHub that the agent didn't find.
  • Gumroad: API exists but company deprecated some endpoints in 2024; agent may overstate current breadth.
